# Журнал изменений RiftLink

Формат: версия **прошивки и приложения** синхронизированы по семантике релиза (например `1.5.19`).

---

## 1.5.19 — 2026-03-23

### Надёжность личных сообщений и BLE

- **Входящие сообщения (`evt:msg`)** — исправлена дедупликация в приложении: при отсутствии стабильного `msgId` в JSON разные сообщения с одинаковым текстом больше не схлопываются; при валидном `msgId` повторная доставка по-прежнему отфильтровывается. Ограничен рост кэша ключей дедупа при длительной сессии.
- **`getInfo`** — при троттлинге запроса повторно выдаётся последний снимок `info` в шину событий, чтобы экраны (соседи, настройки) не «застывали», если свежий JSON уже есть в памяти; при **полностью недоступном транспорте** (нет BLE/Wi‑Fi) в шину снова отдаётся последний закэшированный `evt:info`.
- **Экран чата** — при возврате приложения с фона список сообщений перечитывается из локальной БД, чтобы устранить расхождение «в базе есть — в открытом чате нет»; то же при **возврате с вложенного экрана** поверх чата (например диалог), через глобальный `RouteObserver`.
- **Отправка в личный чат** — при невалидном ID получателя (не 16 hex) в лог добавляется явная причина `invalid_recipient` вместо немого `send() == false`.
- **Ping / проверка «онлайн»** — исправлена логика `sendPing`/`sendPingTracked`: раньше признак успеха почти всегда был «да», даже если BLE не принял JSON (ошибка глоталась); теперь коротко ждём ответ роутера и отличаем `request_send_failed` от ожидания pong по эфиру. После подключения `getInfo` → `getGroups` → `getRoutes` выполняются **по очереди**, чтобы реже переполнять очередь команд на узле. Авто-ping при входе в личку задержан (~520 ms). В списке чатов: таймаут ожидания pong увеличен до 20 s, сопоставление ответа с ожидаемым ID — с учётом префикса (как в экране чата).
- **Traceroute / selftest** — увеличены таймауты ожидания ответа по BLE (`traceroute` 18 s, `selftest` 15 s).
- **Диагностика** — дополнительная строка лога для цепочки прошивка → ingest → репозиторий по входящим сообщениям.

### Тесты и инфраструктура

- Тесты на ключ дедупа входящих сообщений; исправлены ожидания в тестах навигации (дублирование заголовка на тестовой странице).

### Веб-прошивка (ESP Web Flasher)

- Обновлены манифесты и образы `*_full.bin` для Heltec V3, V3 Paper и V4.
